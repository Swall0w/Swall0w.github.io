Setup a directory structure for TDD with Python

To develop your project efficiently, it is know that "Test Driven Development(TDD)" is good way because it make the quality of the project better.

In this post, I'll write a brief introduction of TDD, and how to apply TDD to a python project in a better way.

What is TDD?

In the traditional software development procedure, there are some step; model it, develop it, run tests over it and refactor the project's code. In this approach writing test code is after step of writing code. Therefore, you cannot discover some errors or missing cases until the whole code has been developed. In other words, you may need to spent much time to fix the project code because it is difficult to find bugs in big code. On the contray, Test Driven Development is an innovative development approach which tests are written, before writing code. With TDD, you write test code for functions which does not exist. Then, you write your tagert fucntions until passing their coresspoinding tests.
